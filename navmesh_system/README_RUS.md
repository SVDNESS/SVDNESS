# NavMesh система — Aion Revenge 4.8

Система навигационных сеток для корректного обхода препятствий NPC и мобов на основе гео-данных сервера и библиотеки [recast4j](https://github.com/ppiastucki/recast4j).

---

## Как это работает

### 1. Сборка .nav файлов (SVD-NavMesh-Builder)

Отдельный offline-инструмент читает `.geo` файлы игровых карт, собирает геометрию местности и объектов, и строит навигационные сетки с помощью алгоритма Recast. Результат сохраняется в `.nav` файлы с кастомным заголовком `SVNM`, содержащим метаданные карты, параметры агента и CRC исходной геометрии.

```
SVD-NavMesh-Builder
├── builder/   — сборка навмеша (NavMeshBaker, GeometryAssembler)
├── model/     — модели данных (WorldGeometry, TerrainData, MeshModel)
├── reader/    — чтение .geo файлов (GeoFileReader, TerrainReader)
└── writer/    — запись .nav файлов (NavFileWriter)
```

### 2. Загрузка на сервере (NavMeshService)

При старте сервера `NavMeshService` параллельно загружает все `.nav` файлы из `data/navmesh/`. Каждый файл привязан к `mapId`. Если параметры агента в файле не совпадают с конфигом — сервер выводит предупреждение и предлагает пересобрать `.nav`.

### 3. Поиск пути (Detour)

При движении NPC к цели `NpcMoveController` запрашивает у `NavMeshService` маршрут обхода препятствий. Система конвертирует координаты Aion (ось Z — вверх) в систему Recast (ось Y — вверх), строит путь через `NavMeshQuery` и возвращает очередь промежуточных точек.

```
moveToPoint(x, y, z)
    └── moveViaNavMesh()
            ├── NavMeshService.findPath() → List<Vector3f>
            ├── Очередь waypoints → LinkedList<Point3D>
            └── NPC идёт по точкам, не проходя сквозь стены
```

### 4. Поведение при недостижимости цели

- Если цель внутри непромешенной геометрии — NPC доходит до ближайшей достижимой точки и ждёт там, не проходя сквозь стены.
- Если навмеша нет для карты — fallback на прямолинейное движение.
- Принудительные перемещения (скриптовые, механические) всегда идут по прямой.
- Если цель в радиусе атаки, но за стеной — NPC не стоит на месте до сброса цели,
  а строит маршрут обхода (в дверь, на трап) и продолжает преследование.
- Потеря прямой видимости во время погони — штатная ситуация при включённом NavMesh:
  NPC обходит угол/стену и догоняет цель. От бесконечной погони защищает
  дистанция и таймеры (`checkGiveUpDistance`).
---

## Статистика

- **174 карты** покрыты навигационными сетками
- Загрузка выполняется параллельно при старте сервера
- `NavMeshQuery` создаётся на каждый запрос — полная потокобезопасность
- Троттлинг перезапроса пути при движущейся цели: 500 мс
